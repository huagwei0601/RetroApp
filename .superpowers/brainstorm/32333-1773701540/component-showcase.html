Should we include a Win98 Component Showcase page?

The project needs a component library — the question is whether to include a visual test page alongside it.

A

Components only (production-focused)

Build the 12 Win98 components directly inside the app structure. No separate demo page. Test visually by running the desktop app itself.

Pros

  • Leaner project
  • Matches CLAUDE.md spec exactly

Cons

  • Harder to inspect individual components in isolation
B

Components + /showcase page

Add a /showcase route that renders all 12 Win98 components in isolation — a lightweight Win98-styled storybook. Useful while building, can be hidden later.

Pros

  • See all components at once
  • Fast visual iteration
  • Portfolio-worthy demo page

Cons

  • One extra file to build
